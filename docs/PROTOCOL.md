# 公开协议与数据边界

## 公开默认

公开默认构建不启动 Wi-Fi/TCP，桌面端也不监听任何网络端口。因此默认没有可用的网络协议、远程控制、设备注册、认证、命令或确认语义。

## 可选单向教学遥测

仅当使用者在被忽略的 `hardware/firmware/src/config.local.h` 中将 `ENABLE_EXPERIMENTAL_WIFI_TCP` 设置为精确值 `1`，并给出非空的本机 Wi-Fi 与目标配置，固件才尝试连接受信任的局域网并向指定目标写一行 UTF-8 JSON：

```json
{"temperature":25.1,"humidity":60.0,"light":1500,"co2":820,"pressure":1013.2}
```

字段只是当前代码读取到的数值或零值回退；它们不是经过校准、完整、准确、实时、可靠、适宜或安全的环境结论。写入调用返回也不表示目标接收、持久化、显示、设备在线或有人处理。

### 明确不存在的功能

公开代码**没有**：

- TCP listener、双向会话、设备 ID、配对、设备绑定、认证、授权、TLS、加密、签名、重放防护、审计、速率限制；
- 下行命令、阈值同步、风扇/补光/告警控制、命令 ID、ACK、超时确认、失败状态或可靠投递；
- 远程部署、端口开放说明、公共网络支持或生产运维方案。

若自行修改项目恢复这些能力，必须重新做协议、安全、电气、真机与文档审查，不能沿用本仓状态或构建结论。

## 桌面端本地导入解析器

`desktop/Services/TelemetryParser.cs` 只在代码层解析不超过 4096 字节的单个 JSON 对象，供本地教学展示使用，**不会**打开 socket。公开 UI 当前未提供文件/网络导入按钮。解析成功也只表示 JSON 结构被读取，不表示设备、来源、身份、时间、数据准确性或实时性已验证。
