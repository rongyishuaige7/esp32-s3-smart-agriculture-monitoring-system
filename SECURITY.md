# 安全、隐私、网络与硬件边界

## 不适用场景

本项目不适用于农业自动化、可靠告警、作物健康/适宜性判断、空气质量/安全判断、远程控制、无人值守、应急响应、门禁、安全系统、生产系统或任何需要可靠执行、数据完整性、电气安全或人员处置保证的场景。

不得根据传感器数值、OLED、桌面界面、数据库、局域网请求、GPIO 状态、CI 或构建，作出环境准确、设备在线、作物适宜、实体负载已动作、消息送达、有人处理或安全的结论。

## 网络与控制边界

- 公开桌面端没有 TCP listener、网络遥测接收、下行命令、设备控制页或阈值同步机制。
- 公开默认固件不连接 Wi-Fi/TCP。`ENABLE_EXPERIMENTAL_WIFI_TCP=1` 只会编译本机配置驱动的**单向**教学遥测；它不实现 TLS、设备身份、配对、认证、授权、访问控制、签名、重放防护、审计、命令 ID、ACK、重试成功保证、速率限制或远程控制。
- `ENABLE_EXPERIMENTAL_ACTUATORS=1` 只用于受监督低压台架的 compile-only 分支，公开代码仍使已知风扇信号保持低电平、灯带输出黑色；它不代表驱动极性、外部电路、实体负载、安全或动作结果。
- 禁止公网暴露、端口转发、公共 Wi-Fi、共享热点或不可信网络。项目不提供生产网络、远程管理或可靠协议方案。

## 数据库与隐私边界

- `config.local.h` 被 Git 忽略，不能提交、截图、录屏、粘贴到 Issue 或写进日志。
- 公开桌面端不含账号、数据库、持久化、恢复、权限、审计、备份、删除或生产数据治理能力。
- 不要将真实环境数据、网络信息或客户资料提交到公开仓或问题单。

## 电气与传感器边界

- 公开默认将 GPIO5、GPIO6、GPIO38 设为输入，不代表外部负载已物理关闭、无误动作或电气安全。I²C、DHT11 和 OLED 路径仍可能初始化或产生 I/O。
- GPIO 不得直接驱动风扇、灯带或其他高电流/市电负载。真实台架需独立低压电源、合规驱动、续流/限流、保护、共地和人工监督。

## 问题报告

仓库不承诺私密报告通道。只提交最小化且已脱敏的复现材料；不要上传 Wi-Fi 凭据、私网 IP/MAC、数据库配置/导出、真实环境数据、位置、照片 EXIF/GPS、串口日志、网络抓包、客户资料或预构建产物。
